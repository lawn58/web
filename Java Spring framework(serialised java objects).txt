Для эксплуатации уязвимости необходима утилита https://jitpack.io/com/github/frohoff/ysoserial/master-SNAPSHOT/ysoserial-master-SNAPSHOT.jar
При получении cookie мы должны заметить то, что куки начинается с rO0 , это хорошая заметка о том, что приложение уязвимо к serialised Java objects.
Итак, что проэксплуатировать, нам необходимо следующее:
1) запустить приложение с параметром Spring1, сформировать полезную нагрузку и закодировать в base64 без переноса строк, вывод помещаем в текстовый файл, после чего содержимое файла вставляем в поле Cookie: auth=
./ysoserial-master-SNAPSHOT.jar Spring1 "/usr/bin/uname" | base64 -w 0 > file.txt
2) после чего отправляем на сервер и должны увидеть вывод команды uname.
